## CSV / 表格数据处理经验

（棘轮原则：以下每条都对应一类真实观察到的 agent 失败模式）

1. **动手前先看数据**：用 `bash` 跑 `head -5 文件名` 和 `wc -l 文件名` 确认表头、
   列结构、行数，再决定方案。失败模式：凭任务描述想象列结构，算错列。
2. **列序号按自然语言习惯是 1-based**："第三列"指人类数的第 3 列（含表头那行的
   第 3 个字段），不是下标 2 的"第三个索引"混乱。先打印表头逐列确认。
3. **大于几百行就别用 read_file 全量读**：读不完且浪费上下文。直接用一条
   `python -c` 或 `awk` 流式计算，例如：
   `python -c "import csv; vals=[float(r[2]) for r in list(csv.reader(open('data.csv',encoding='utf-8')))[1:] if r[2].strip()]; print(sum(vals)/len(vals), len(vals))"`
4. **脏数据默认剔除而非按 0 计**：空串、N/A、非数字的单元格应排除在分子和分母
   之外，并在最终答复中报告剔除了多少条。失败模式：无效值当 0，均值被拉低。
5. **交叉验证再交付**：用 calculator 或第二种方法（如 awk 对比 python）重算一遍
   关键数字，两种途径一致才写结果文件。失败模式：单一路径的静默计算错误。
6. **结果文件只写要求的内容**：要求"写均值"就只写数字（必要时一行），不要附加
   解释文字——解释放在最终答复里。写完用 read_file 读回确认。
7. **文件名对不上就先 list_files**：任务描述的文件名和实际不符时，列目录找最
   接近的文件并在答复中说明，不要臆造数据。
